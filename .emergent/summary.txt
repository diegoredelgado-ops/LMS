<analysis>**original_problem_statement:**
Build a full-stack Learning Management System (LMS) scalable for 500 concurrent users with the following requirements:

**Core Functionality:**
- A complete educational platform with three user roles: Admin, Instructor, and Student.
- Authentication system with email/password and GitHub OAuth login.
- User profiles with photos, names, and bios.

**Course Management:**
- Instructors and admins can create, edit, and delete courses.
- Each course needs: title, description, cover image, category, level (beginner/intermediate/advanced), price (free/paid simulation), and a structured list of lessons.
- Drag-and-drop lesson ordering for instructors.

**Lesson Types:**
- Video lessons (YouTube/Vimeo embedding).
- Rich text lessons (with a simple editor).
- File-based lessons (PDFs and other documents).

**Student Experience:**
- Course enrollment system (immediate for free courses, simulated checkout for paid).
- Progress tracking per course: completed lessons, percentage complete.
- Students can mark lessons as completed.
- Optional: comments or questions on each lesson.

**Quizzes (nice to have):**
- Instructors can add multiple-choice quizzes to courses.
- Students receive scores upon completion.

**Dashboards by Role:**
- Admin dashboard: user management, platform statistics, course oversight.
- Instructor dashboard: their courses, student enrollment stats, average progress.
- Student dashboard: courses in progress, completed courses, recommendations.

**Technical Requirements:**
- Use Supabase for PostgreSQL database, authentication, and file storage.
- Implement proper Row Level Security (RLS) policies for data protection.
- Connect GitHub for OAuth authentication.
- Build for 500 concurrent users: implement pagination, optimized queries, and caching.
- Responsive modern design with dark/light mode option.
- Accessible UI components.

**Key Technical Concepts**
- **Backend:** FastAPI, SQLAlchemy, PostgreSQL, Alembic (for migrations).
- **Frontend:** React, Tailwind CSS, Framer Motion, Axios.
- **Authentication:** JWT for email/password, OAuth for GitHub.
- **Database:** Supabase (PostgreSQL).
- **AI Integration:**  library to connect with OpenAI via Emergent LLM Key.

**key DB schema**
- **User:** {id, email, hashed_password, full_name, role, github_id, created_at, updated_at}
- **Course:** {id, title, description, instructor_id, category, level, price, published, created_at, updated_at}
- **Lesson:** {id, title, content, lesson_type, video_url, course_id, order, created_at, updated_at}
- **Enrollment:** {id, student_id, course_id, enrolled_at}
- **Quiz:** {id, course_id, title}
- *Other models exist for Questions, Choices, Submissions, etc.*
